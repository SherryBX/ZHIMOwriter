# ZHIMOwriter

一个面向微信公众号排版场景的 Markdown 编辑与预览工作台。

![ZHIMOwriter Cover](./.github/assets/repo-cover.png)

> 左侧写 Markdown，右侧实时预览公众号排版，整理完成后可直接复制到公众号编辑器。

## 项目亮点

- 双栏工作台：中间直接编写 Markdown，右侧同步查看公众号文章效果
- 更贴近公众号排版：标题、加粗、引用、代码块、图片、图片说明都做了定制化渲染
- 图片支持更实用：支持本地绝对路径图片，也支持图床远程图片
- 复制链路可用：复制到公众号时尽量保留样式、图片与排版结构
- 代码块更稳：预览区保留良好展示，复制到公众号后会自动换行，避免长代码撑坏版式

## 适合用来做什么

- 公众号文章初稿整理
- 产品发布手册
- 技术教程与实践记录
- 知识卡片类内容排版
- 需要先在本地编辑、再复制进公众号后台的图文稿件

## 当前能力

- 直接输入 Markdown
- 导入 `.md` 文件
- 实时预览公众号样式
- 支持标题、段落、列表、引用、分隔线、表格、代码块
- 支持 `highlight.js` 代码高亮
- 支持本地绝对路径图片渲染
- 支持图床远程图片渲染
- 支持图片说明文字显示在图片下方
- 支持复制 HTML 内容到公众号编辑器

## 图片支持说明

当前支持以下几类图片地址：

- Windows 本地绝对路径
  例如：`D:/Typora/IMG/demo.png`
- `file:///` 形式的本地文件地址
- 图床远程链接
  例如：`https://your-cdn.com/demo.png`

项目内部对本地图片做了额外处理，让浏览器预览阶段也能正确加载，不会直接被 `file://` 安全策略拦住。

## 使用方式

1. 左侧输入 Markdown，或者点击顶部按钮导入 `.md`
2. 在右侧查看公众号排版预览
3. 确认样式无误后，点击“复制到公众号”
4. 回到微信公众号后台粘贴内容

## 本地启动

```bash
npm install
npm run dev
```

默认启动后即可在本地浏览器中打开编辑器页面。

## 构建

```bash
npm run build
```
