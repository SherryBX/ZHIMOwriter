# 微信公众号模板编辑器骨架设计

**日期：** 2026-03-30

## 目标

搭建一个可扩展的微信公众号模板站点首版前端骨架，首页直接作为编辑器入口。首版完成 Markdown 写作工作台、导入 `.md` 文件入口和实时公众号预览，不接入后端、登录、发布能力，也不锁定最终视觉主题。

## 范围

本次只实现以下能力：

- 顶部操作栏
- 左侧模块工具栏
- 中间 Markdown 编辑区
- 右侧公众号预览区
- 基于 Markdown 内容的实时渲染
- 导入本地 `.md` 文件
- 便于后续替换视觉风格的主题变量层

本次不实现以下能力：

- 真实保存、发布、导出
- 登录与账号体系
- 后端接口
- 拖拽排序
- 完整配置面板，只保留后续扩展接口

## 技术方向

- 前端框架：React
- 构建工具：Vite
- 语言：TypeScript
- 样式：原生 CSS + CSS 变量

选择这一组合的原因是启动快、结构轻、后续替换样式成本低，适合作为编辑器骨架的第一版。

## 页面结构

首页采用单页编辑器布局，由以下区域组成：

1. `AppHeader`
   负责站点标题、当前文稿名、导入 `.md`、保存/发布按钮和预留视图操作入口。
2. `BlockLibrarySidebar`
   作为写作辅助侧栏，展示公众号常用结构提示和排版建议，而不是内容卡片堆叠区。
3. `EditorWorkspace`
   作为 Markdown 主编辑区，提供大尺寸文本输入区域，支持粘贴和导入文件后的继续编辑。
4. `PreviewPanel`
   模拟微信公众号文章阅读视图，按移动端文章宽度实时渲染 Markdown 内容。

## 数据模型

页面核心状态切换为 Markdown 字符串：

```ts
type EditorDocument = {
  title: string;
  markdown: string;
};
```

预览区通过轻量 Markdown 解析器把字符串转换成结构化节点。首版支持：

- `#`、`##`、`###` 标题
- 普通段落
- `>` 引用
- `-` 无序列表

这样可以先满足公众号写作主流程，后续再按需要补图片、分割线、按钮卡片等扩展语法。

## 组件边界

- `App.tsx`：挂载页面级编辑器容器
- `components/editor-shell/*`：编辑器各区域组件
- `data/mockMarkdown.ts`：首版 Markdown 示例内容
- `utils/markdown.ts`：Markdown 解析逻辑
- `styles/*`：全局样式、主题变量和组件布局

## 视觉方向

首版风格贴近参考图，但更明确地偏向内容写作工具：

- 浅色背景与柔和阴影
- 更大的主工作区占比
- 更克制的圆角和更稳的边界
- 接近真实写作产品的编辑器界面气质
- 预览区突出“手机文章卡片”感

视觉上只建立框架感，不在本轮绑定具体品牌风格。

## 响应式要求

- 桌面端保持左中右三栏，但中间编辑区明显大于左右两侧
- 窄屏下右侧预览区下移，避免内容不可用
- 各面板保持最小可读宽度

## 扩展点

- Markdown 解析层，支持后续新增卡片、时间轴、对话块等扩展语法
- CSS 变量主题层，支持后续切换公众号模板风格
- 预留 `InspectorPanel` 接口，供后续模块配置使用
- 文档结构与样式解耦，便于后续接入保存、模板市场和导出

## 验收标准

- 项目能在本地启动
- 首页可展示完整编辑器骨架
- 左侧辅助栏、中间 Markdown 编辑区、右侧公众号预览同时可见
- 输入 Markdown 后预览可实时同步
- 支持导入本地 `.md` 文件并显示内容
- 样式结构清晰，可继续叠加具体模板风格
